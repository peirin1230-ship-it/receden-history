最終更新: 2026-07-14(URL・件数はこの時点で確認)

## 1. 提供元

- **社会保険診療報酬支払基金「基本マスター」**(本プロジェクトの入手元)
  https://www.ssk.or.jp/seikyushiharai/tensuhyo/kihonmasta/
  全件ファイルに加え、更新日ごとの改定分ファイル(CSV)+改定分内容(PDF)を提供。
  過去世代(平成24年〜)のアーカイブページあり。
- **診療報酬情報提供サービス**(厚労省) https://shinryohoshu.mhlw.go.jp/
  全件ファイルは支払基金掲載分と同一。こちらから取得しても可。

## 2. 世代(スナップショット)一覧

`config/eras.yaml` と同期させること。施行日は点数表(本体)の施行日。

| era | 改定 | 施行日 | 対象マスター | 備考 |
|---|---|---|---|---|
| h24 | 平成24年度 | 2012-04-01 | 全部 | ベースライン(これ以前の履歴は復元不能) |
| h26 | 平成26年度 | 2014-04-01 | 全部 | 消費税8%対応を含む |
| h28 | 平成28年度 | 2016-04-01 | 全部 | |
| h30 | 平成30年度 | 2018-04-01 | 全部 | |
| r01 | 令和元年 | 2019-10-01 | 全部 | 消費税10%引上げ対応の臨時改定(短い世代) |
| r02 | 令和2年度 | 2020-04-01 | 全部 | |
| r04 | 令和4年度 | 2022-04-01 | 全部 | |
| r06 | 令和6年度 | 2024-06-01 | 全部 | **本体改定の施行が6月1日に後ろ倒しされた**(4月ではない) |
| r07 | 令和7年度薬価改定 | 2025-04-01 | **Y のみ** | 医薬品のみの中間年改定。`masters: [Y]` |
| r08 | 令和8年度(現行) | 2026-06-01 ※要確認 | 全部 | 現行ページから取得。4/1施行の項目もあり得るため、改定分内容PDFで施行日を確認して eras.yaml を確定させる |

各世代アーカイブの全件ファイルは、基本的に**その世代の最終状態**
(例: h24 の全件 ≒ 2014年3月時点)。世代の途中経過は改定分ファイルにのみ残る(Phase 2 対象)。

### 医薬品(Y)と薬価改定について

- 薬価(医薬品の金額)は診療報酬本体と別に**毎年**改定される(中間年薬価改定:
  令和3・5・7年度)。令和7年度分のみ独立した世代 r07 として収載している
  (r06 世代内に全件ファイルが2つ存在するため)。令和3・5年度分の変更は
  r02・r04 の世代内変更として、レコードの変更年月日から exact の日付で復元される。
- r06 以降は本体(6/1)と薬価改定(4/1)の施行日がずれる。このため r06・r08 には
  `effective_date_overrides: {Y: 4/1}` を設定し、医薬品の期間窓・世代境界は
  薬価改定日(4/1)基準で計算される(設定しないと、変更年月日=4/1 の大量の変更が
  「窓外」と判定されて era_boundary に化ける)。

## 4. ファイル名の規約

- 現行世代の命名: 医科 `s_ALLyyyymmdd.zip`(中身 `s_ALLyyyymmdd.csv`)、
  特定器材 `t_ALL...`、コメント `c_ALL...`(マスター種別文字の小文字が接頭辞)
- **過去世代では命名が異なる可能性がある**。リネームは不要。
  ツールはファイル名に依存せず、CSVの2列目(マスター種別 S/T/C)で自動判定する
- 1つの era ディレクトリに同一マスターのCSVが複数あるのは不可(validate でエラー)

## 5. 配置ルール

```
data/raw/
├── h24/   ← h24ページの 医科・医薬品・特定器材・コメント の全件CSV(4ファイル)
├── h26/
├── ...
├── r07/   ← 医薬品のみ(令和7年度薬価改定の全件CSV 1ファイル)
└── r08/   ← 現行ページの4ファイル
```

- ZIPは解凍してCSVを置く(ZIP直置き対応は必須ではない)
- ダウンロードした元ZIPを保管したい場合は `data/raw/{era}/zip/` 等に置いてよい
  (ツールは `*.csv` のみ読む)

## 7. 生データのGit管理・再配布に関する注意

- 基本マスターは公的に無償公開されているが、**公開リポジトリに生データを同梱する前に**
  支払基金サイトの利用条件( https://www.ssk.or.jp/riyo.html )を確認すること
- 迷う場合の安全策:
  - リポジトリを private にする、または
  - `data/raw/` を .gitignore し、本ドキュメントのチェックリスト+ダウンロード手順で再現する
- どちらの場合も、取込済みファイルの sha256 を snapshots テーブルに記録して
  再現性を担保する(REQUIREMENTS.md F2)
- **GitHub Pages で公開するサイトは(Enterprise の限定公開を除き)誰でも閲覧できる**。
  private リポジトリであっても Pages サイト自体は公開になる点に注意。
  サイトに載せる派生データの扱いは `docs/SITE_SPEC.md` §5〜6 の出典・免責方針に従うこと
