# ファイルレイアウト: 全件マスターCSVの仕様とキーカラム

出典: レセプト電算処理システム マスターファイル仕様説明書(令和8年版)
「ファイルレイアウト」(master_3_20260630.pdf) を 2026-07-14 に確認して転記。
**列位置の一次確認が取れているのは現行(令和8年版)レイアウトのみ**(§5 参照)。

## 1. 全マスター共通仕様

- **ヘッダ行なし**。1行目からデータ
- 区切り: カンマ(`，`表記だが実体は ASCII カンマ)
- 文字コード: **cp932**(Shift_JIS系。①等の拡張文字があるため cp932 指定)
- 改行: CRLF
- 数字モードは**前ゼロ省略**。日付が未設定の場合は `0`
- 予備列は `0`
- 引用符の有無は世代・マスターにより異なる可能性 → csv モジュールで両対応
- 日付列(変更年月日・廃止年月日等)は**西暦8桁 YYYYMMDD**
- 「項番」= CSV列位置(1始まり)。繰り返し項目も1繰り返し=1列で採番されている

### 変更区分(全マスター共通・第1列)

| 値 | 意味 |
|---|---|
| 0 | 前回マスターの内容と同じ |
| 1 | 抹消 |
| 3 | 新規 |
| 5 | 変更 |
| 9 | 廃止(当該コードが使用不可になったことを表す。**次回マスター更新時にレコード削除**) |

## 2. 医科診療行為マスター(マスター種別 S)

- 現行レイアウト: **全150列**。コード9桁
- 繰り返し項目: 施設基準①〜⑩ = 列72〜81、年齢加算①〜④(3項目×4) = 列100〜111

### キーカラム(本ツールが使う列)

| 列 | 項目名 | 用途 |
|---|---|---|
| 1 | 変更区分 | 異動種別 |
| 2 | マスター種別 | `S` 固定(ファイル自動判定に使用) |
| 3 | 診療行為コード | **主キー**(9桁) |
| 5 | 診療行為省略漢字名称 | 表示・差分対象 |
| 7 | 省略カナ名称 | 参考 |
| 11 | 点数識別 | 点数(列12)の解釈区分。値の意味は仕様説明書別紙参照 |
| 12 | 新又は現点数 | **差分対象**(点数改定の検出) |
| 59 | 漢字名称変更区分 | 名称変更フラグ(参考) |
| 60 | カナ名称変更区分 | 参考 |
| 87 | **変更年月日** | イベント日付の推定(YYYYMMDD / 0) |
| 88 | **廃止年月日** | 廃止日=使用期限(YYYYMMDD / 0) |
| 89 | 公表順序番号 | 並び順 |
| 113 | 基本漢字名称 | 正式名称。**差分対象** |
| 117 | 点数表区分番号 | 点数表上の区分(例: A000系)。世代跨ぎの同定補助 |

注意: **収載年月日(新設日)の列は存在しない**。

## 4. コメントマスター(マスター種別 C)

- 現行レイアウト: **全30列**
- コード体系: コメントコード9桁 = `8`(区分固定) + パターン2桁 + 一連番号6桁

### キーカラム

| 列 | 項目名 | 用途 |
|---|---|---|
| 1 | 変更区分 | 異動種別 |
| 2 | マスター種別 | `C` 固定 |
| 3 | 区分 | `8` 固定 |
| 4 | パターン | コメント文への付加情報形式(詳細は仕様説明書) |
| 5 | 一連番号 | 前ゼロ省略に注意 |
| 7 | 漢字名称(コメント文) | 表示・**差分対象** |
| 10〜17 | レセプト編集情報①〜④(カラム位置/桁数 ×4) | 参考 |
| 20 | 選択式コメント識別 | **差分対象**(選択式コメント化の検出) |
| 21 | **変更年月日** | イベント日付の推定 |
| 22 | **廃止年月日** | 廃止日 |
| 23 | **コメントコード** | **主キー**(9桁)。※§5の注意参照 |
| 24 | 公表順序番号 | 並び順 |

### コメントコードの導出(列23が無い/空の世代への対応)

```
コメントコード = "8" + str(パターン).zfill(2) + str(一連番号).zfill(6)
```

列23(コメントコード)は後年のレイアウトで追加された可能性があるため、
存在しない・空の場合は上式で導出する。存在する場合は導出値と一致することを
validate で確認する。

## 5. 世代間のレイアウト差異について(重要)

- 上記の列位置は**令和8年版仕様説明書で確認した現行レイアウト**。
  `config/layouts/*.yaml` では `verified: true` として定義済み
- 過去世代(h24〜r06)は「項目追加は末尾追加・予備列の転用が基本」という慣行から
  **主要列(特に前半の列と変更年月日・廃止年月日)は同位置と期待できるが未検証**。
  layouts では暫定的に現行と同位置(`verified: false`)としてある
- 実装時の確認手順:
  1. `validate` の列プローブ(コード列 `^\d{9}$` 率、日付列 `^(0|\d{8})$` 率、
     名称列の非ASCII率)を世代×マスターごとに実行
  2. ズレが検出された世代は、その世代の仕様説明書
     (`docs/DATA_SOURCES.md` §6 の各世代「変更情報」ページから入手)で列位置を確認し、
     `config/layouts/` に世代別エントリを追加して `verified: true` にする
  3. 実測列数は snapshots テーブルに記録し、README に世代×マスターの列数表を残す
- 列数は世代とともに増えている想定(現行: S=150 / T=38 / C=30)。
  読み込んだ行の列数が layout の想定より少ない場合、**末尾列は「存在しない」として
  扱い(エラーにしない)**、差分判定から除外する(REQUIREMENTS.md §6.3)
